## 문제3

```
computed vs watch
- Computed: 
계산해야 하는 목표 데이터를 정의하는 방식으로 SW에서 얘기하는 '선언형 프로그래밍' 방식
JS 표현식 결과를 미리 계산해서 저장(캐싱)
methods와 형태는 같지만 캐싱 여부의 차이가 있다.
입력값이 같은 경우, 데이터의 변화에 따라 계산된 값을 화면에 출력해야 하는 경우
계산된 값을 저장하고 리턴이 되면 그 값을 넘겨줌

- watch: 
감시할 데이터를 저장하고 해당 데이터가 변경되면 ~한 함수를 실행하라는 방식으로 SW에서 얘기하는 '명령형 프로그래밍' 방식
(Vue) instance의 데이터 변경을 관찰(보고)하고 있다가 Data Object에서 특정한 데이터의 변화가 감지되면 함수를 실행(localStorage)
비동기 처리 로직이 필요한 경우(요청과 응답의 시간이 지금 당장 결정되어 있지 않은 경우), 캐싱(저장)하지 않고 매번 출력값을 바꿔야 하는 경우
vue 인스턴스의 data 변화(경)를 관찰하고 이에 반응
데이터 변경에 대한 응답으로 비동기식 또는 시간이 많이 소요되는 조작을 수행할 때 적합
```
